Copilot AI Credits 预算管理

User-Level Budgets（用户级预算）· 纯前端调用 GitHub REST API · PAT 仅存于浏览器内存

重要前提：本页使用 Enterprise 级别预算 API。PDF 同时记录了 Organization 级别路径；若要操作 org，需要把路径替换为 /organizations/{org}/settings/billing/budgets 并使用相应权限。

前置条件清单（请逐项自查）

  • 使用 Enterprise 账户（Copilot Enterprise / Business / Standalone），或确认要改用 Organization 级别路径。
  • 已在 Settings > AI Controls > Copilot 开启 "Premium request paid overages"。
  • 已移除任何现存的 $0 企业级 premium request 预算（它会优先生效并阻断所有人）。
  • PAT 具备预算 API 所需 scope（PDF 推荐 manage_billing:copilot；org 场景也可使用 admin:org）。
  • 操作者为 Enterprise Owner 或 Billing Manager 角色。

核心概念与优先级

Cost Center
成本中心，用于归集计费的逻辑分组。
Budget（预算）
对 Copilot AI credits 用量设置的金额上限，达到上限后可阻断后续用量。
Universal 预算（budget_scope: "multi_user_customer"）
面向所有 Copilot 用户的默认预算。每个企业只能有一个 universal 预算，重复创建返回 409。
个人预算（budget_scope: "user"）
面向单个 GitHub 用户的预算。个人预算始终优先于 universal 预算。
Enterprise 预算（budget_scope: "enterprise"）
面向整个 enterprise 的 metered 消费总上限。
user scope 会覆盖 universal 预算；Enterprise 预算用于总池上限。每位用户的 included usage 先消耗，consumed_amount 只反映超出部分的金额。

参数含义

PAT
个人访问令牌，需具备预算 API 所需 scope。仅保存在浏览器内存中。
enterprise（企业 slug）
企业标识符，用于构造 API 路径。
budget_amount
预算金额（美元）。设为 0 表示默认阻断全部 premium 用量。
budget_scope
multi_user_customer（universal）、user（个人）或 enterprise（企业总上限）。
budget_product_sku
Copilot 新计费模型使用 ai_credits。
budget_entity_name
预算所属实体名称。user scope 创建时需与 user 字段同值。
user
仅 user scope 需要，填 GitHub 用户名（无需数字 ID）。

所有请求固定带头：Accept: application/vnd.github+json、X-GitHub-Api-Version: 2026-03-10、Authorization: Bearer <PAT>。

API 使用示例（curl）

以下示例覆盖本页实际用到的预算 API。请先设置环境变量：export PAT=<你的令牌>、export ENT=<企业 slug>、export USER=octocat、export BUDGET_ID=<预算 id>。

创建预算页里的 upsert 逻辑是：先用 list API 按 scope 查询已有预算；如果找到已有预算 id，则 PATCH；如果未找到，则 POST 创建。

① 列出全部预算

curl \
  -H "Accept: application/vnd.github+json" \
  -H "X-GitHub-Api-Version: 2026-03-10" \
  -H "Authorization: Bearer $PAT" \
  "https://api.github.com/enterprises/$ENT/settings/billing/budgets?per_page=100"

② 列出个人预算（scope=user）

curl \
  -H "Accept: application/vnd.github+json" \
  -H "X-GitHub-Api-Version: 2026-03-10" \
  -H "Authorization: Bearer $PAT" \
  "https://api.github.com/enterprises/$ENT/settings/billing/budgets?scope=user&per_page=100"

③ 列出 Universal 预算（scope=multi_user_customer）

curl \
  -H "Accept: application/vnd.github+json" \
  -H "X-GitHub-Api-Version: 2026-03-10" \
  -H "Authorization: Bearer $PAT" \
  "https://api.github.com/enterprises/$ENT/settings/billing/budgets?scope=multi_user_customer&per_page=100"

④ 列出 Enterprise 总预算（scope=enterprise）

curl \
  -H "Accept: application/vnd.github+json" \
  -H "X-GitHub-Api-Version: 2026-03-10" \
  -H "Authorization: Bearer $PAT" \
  "https://api.github.com/enterprises/$ENT/settings/billing/budgets?scope=enterprise&per_page=100"

⑤ 创建 Universal 用户预算（multi_user_customer）

curl -X POST \
  -H "Accept: application/vnd.github+json" \
  -H "X-GitHub-Api-Version: 2026-03-10" \
  -H "Authorization: Bearer $PAT" \
  "https://api.github.com/enterprises/$ENT/settings/billing/budgets" \
  -d '{
    "budget_amount": 50,
    "budget_scope": "multi_user_customer",
    "budget_entity_name": "'$ENT'",
    "prevent_further_usage": true,
    "budget_product_sku": "ai_credits",
    "budget_type": "BundlePricing",
    "budget_alerting": { "will_alert": false, "alert_recipients": [] }
  }'

⑥ 创建 Individual 用户预算（user scope）

curl -X POST \
  -H "Accept: application/vnd.github+json" \
  -H "X-GitHub-Api-Version: 2026-03-10" \
  -H "Authorization: Bearer $PAT" \
  "https://api.github.com/enterprises/$ENT/settings/billing/budgets" \
  -d '{
    "budget_amount": 20,
    "budget_scope": "user",
    "budget_entity_name": "'$USER'",
    "user": "'$USER'",
    "prevent_further_usage": true,
    "budget_product_sku": "ai_credits",
    "budget_type": "BundlePricing",
    "budget_alerting": { "will_alert": false, "alert_recipients": [] }
  }'

⑦ 创建 Enterprise 总预算（enterprise scope）

curl -X POST \
  -H "Accept: application/vnd.github+json" \
  -H "X-GitHub-Api-Version: 2026-03-10" \
  -H "Authorization: Bearer $PAT" \
  "https://api.github.com/enterprises/$ENT/settings/billing/budgets" \
  -d '{
    "budget_amount": 1000,
    "budget_scope": "enterprise",
    "budget_entity_name": "'$ENT'",
    "prevent_further_usage": true,
    "budget_product_sku": "ai_credits",
    "budget_type": "BundlePricing",
    "budget_alerting": { "will_alert": false, "alert_recipients": [] }
  }'

⑧ 查看指定预算（GET by ID）

curl \
  -H "Accept: application/vnd.github+json" \
  -H "X-GitHub-Api-Version: 2026-03-10" \
  -H "Authorization: Bearer $PAT" \
  "https://api.github.com/enterprises/$ENT/settings/billing/budgets/$BUDGET_ID"

⑨ 更新预算（调额，PATCH）

curl -X PATCH \
  -H "Accept: application/vnd.github+json" \
  -H "X-GitHub-Api-Version: 2026-03-10" \
  -H "Authorization: Bearer $PAT" \
  "https://api.github.com/enterprises/$ENT/settings/billing/budgets/$BUDGET_ID" \
  -d '{ "budget_amount": 80 }'

⑩ 删除预算（DELETE）

curl -X DELETE \
  -H "Accept: application/vnd.github+json" \
  -H "X-GitHub-Api-Version: 2026-03-10" \
  -H "Authorization: Bearer $PAT" \
  "https://api.github.com/enterprises/$ENT/settings/billing/budgets/$BUDGET_ID"

公共凭据

① 统一预算（Universal 默认预算）

为所有 Copilot 用户创建默认预算，budget_scope: "multi_user_customer"。每企业仅一个，已存在则更新（PATCH），不存在则创建（POST）。

② 批量差异化预算（个人预算 · upsert）

逐个为指定用户处理个人预算，budget_scope: "user"。已存在则更新（PATCH），不存在则创建（POST）。支持上传 CSV 或粘贴列表，首行表头 user,budget。

③ Enterprise scope 预算（总池上限 · upsert）

用于设置整个 enterprise 的 metered 消费总上限，budget_scope: "enterprise"。已存在则更新（PATCH），不存在则创建（POST）。
